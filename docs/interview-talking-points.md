# Interview Talking Points

Prepare these senior engineering responses to showcase deep architectural mastery during technical rounds.

### 1. Why does one single repository contain four different UI frameworks?
- **Answer**: It simulates a real-world enterprise merger scenario. Large corporate consolidations often inherit disparate technical stacks (e.g. an inherited Angular clinical system, a lightweight Vue operations tool, a legacy jQuery administration console). The platform demonstrates how a Principal Engineer unifies these panels under a single React Host shell and standardizes them using a common Sass design system and shared TypeScript schemas.

### 2. How does the React Host shell federate other modules?
- **Answer**: The React Host shell acts as the orchestrator. It manages user authentication states, session policies, executive analytics graphs (Recharts), and mounts sandboxed live micro-frontend workspace simulators. It details a modern roadmap for deprecating legacy units.

### 3. How does Angular support compliance and complex data?
- **Answer**: The healthcare care manager module handles complex clinical data grids. Angular is suited for this compliance due to its strict object models, dependency injection services, functional route guards, reactive forms validations, and debounced RxJS streams, which handle patient search inputs cleanly.

### 4. What are Vue's advantages in SRE telemetry?
- **Answer**: SRE teams require fast, lightweight dashboards. Vue 3's composition API allows for simple state reactive logic. Telemetry graphs are rendered directly using responsive inline browser SVG structures rather than heavy external libraries, keeping standalone builds small (88KB).

### 5. Why is jQuery included, and how is it modernized?
- **Answer**: Real enterprises still maintain critical jQuery applications due to custom database logic baked into selector listeners. Rather than attempting high-risk rewrite overrides, we isolated its direct queries into structured API adapters (`shared/mock-api/`) and encapsulated it inside our consolidated host shell, providing a safe, incremental modernization path.
